- 스코어링? : 머신러닝의 전단계. 소비자에게 지표를 랭킹을 보여주는 것. 
- 추천 모델, 머신러닝 (번역이 필요하다던지 개인화 추천, 랭킹 모델을 만들어 낼것인지 = 데이터 사이언티스트)
- 위의 해당 모델의 성과분석, 소비자의 반응( 데이터 분석가)




챔피언 티어 어떻게 만들었는가?



제한된 기능: 코드가 없는 도구는 맞춤형 코딩 솔루션에 비해 기능이 제한된 경우가 많습니다. 기본 애플리케이션을 구축하는 데는 적합할 수 있지만 더 복잡한 프로젝트를 지원할 만큼 강력하지 않을 수 있습니다.  
  
제한된 사용자 정의: 코드 없는 도구는 사용자 정의가 가능하지만 사용자가 솔루션을 사용자 정의할 수 있는 정도에는 제한이 있을 수 있습니다. 타사 애플리케이션과 통합하거나 복잡한 로직을 애플리케이션에 구축할 때 특히 그렇습니다.  
  
제한된 확장성: 코드가 없는 도구는 많은 수의 사용자 또는 많은 양의 데이터를 지원하도록 확장하지 못할 수 있습니다. 이는 빠르게 성장하거나 많은 양의 데이터를 처리해야 하는 애플리케이션에 제한이 될 수 있습니다.  
  
플랫폼에 따라 다름: 노코드 도구는 특정 플랫폼에 구축되는 경우가 많습니다. 즉, 사용자는 해당 플랫폼의 기능으로 제한될 수 있습니다. 플랫폼이 변경되거나 사업이 중단되면 사용자는 유지 관리하거나 수정할 수 없는 솔루션을 갖게 될 수 있습니다.  
  
보안 및 개인정보 보호 문제: 코드가 없는 도구를 사용하면 애플리케이션을 쉽게 구축할 수 있지만 사용자는 애플리케이션이 안전하고 개인정보 보호 규정을 준수하는지 확인할 수 있는 기술 지식이 없을 수 있습니다. 이는 민감한 데이터를 처리하거나 비즈니스 목적으로 사용되는 애플리케이션에 문제가 될 수 있습니다.

### 빠르고 쉽고 간편
- 프로그래밍 언어를 몰라도 애플리케이션이나 웹사이트를 만들 수 있는 소프트웨어 플랫폼
- 드래그 앤 드롭 등의 손쉬운 방법으로 사용자가 응용 프로그램을 디자인하고 개발 가능!

### 강력하지만… 강력하지만?
- 제한된 기능
    - 강력하고 편리하지만, 커스터마이징이 힘듦
- 비교적 느린…
    - swift, Kotlin, Java 등 각 OS와 적절히 호환되는 언어로 개발된 네이티브 서비스에 비해서 낮은 성능을 보여줌
- 보안 문제
    - GDPR, HIPAA 또는 PCI DSS와 같은 규정 준수를 자동으로 보장하지 못할 수도
### 그럼에도 불구하고 사용하는 이유
- 전문지식이 없는 사람도 손쉽게 서비스를 구축할 수 있음
- 개발자를 구할 여력이 없는 기업이나 팀도 쉽게 사용할 수 있음
- 때때로 서비스 크기에 따라 개발자를 구하는 것보다 효율적인 가격으로 서비스를 구축할 수 있음
- 사업 검증을 위해 빠른 속도로 핵심 가설을 테스트 해 볼 수 있음! (MVP 제작)
